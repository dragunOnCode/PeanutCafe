## 🚀 快速启动（3步）

### Step 1: 克隆并安装依赖

```bash
# 进入项目目录
cd d:/Project/Lobster/backend

# 安装依赖（首次运行需要3-5分钟）
npm install
```

### Step 2: 启动基础设施

```bash
# 启动PostgreSQL、Redis、ChromaDB、MinIO
cd docker
docker-compose up -d

# 验证所有服务启动
docker-compose ps

# 应该看到4个服务都是 "Up" 状态
```

### Step 3: 配置环境变量并启动

```bash
# 回到backend根目录
cd ..

# 复制环境变量模板
cp .env.example .env

# 编辑.env，至少配置以下关键变量：
# - OPENROUTER_API_KEY=你的OpenRouter API Key
# - GEMINI_API_KEY=你的Gemini API Key（如果使用Gemini）

# 启动开发服务器
npm run start:dev
```

✅ **完成！** 访问 http://localhost:3000/health 应该看到健康检查响应。

## 📝 详细步骤（首次开发）

### 1. 项目初始化

```bash
# 如果是全新开发，需要先创建NestJS项目
npm install -g @nestjs/cli

# 在backend目录初始化项目
cd d:/Project/Lobster/backend
nest new . --package-manager npm --skip-git

# 安装核心依赖
npm install @nestjs/websockets @nestjs/platform-socket.io socket.io
npm install @nestjs/typeorm typeorm pg
npm install @nestjs/config
npm install ioredis
npm install class-validator class-transformer

# 安装开发依赖
npm install -D @types/node prettier eslint
```

### 2. 数据库初始化

```bash
# 确保Docker容器运行
docker-compose ps

# 连接到PostgreSQL
psql -h localhost -p 5432 -U admin -d lobster
# 密码：password

# 创建pgvector扩展
CREATE EXTENSION IF NOT EXISTS vector;

# 退出psql
\q

# 运行数据库迁移（如果已有迁移文件）
npm run migration:run
```

### 3. Redis验证

```bash
# 连接到Redis
redis-cli -h localhost -p 6379

# 测试连接
ping
# 应返回: PONG

# 退出redis-cli
exit
```

### 4. 配置Agent CLI工具

```bash
# 如果使用Codex CLI（假设已安装）
codex-cli --version

# 如果使用Gemini CLI（假设已安装）
gemini --version

# 如果没有安装，可以先跳过，只使用Claude (HTTP API)
```

## 🐛 常见问题

### 1. Docker服务启动失败

**问题**：`docker-compose up -d` 失败

**解决**：
```bash
# 检查端口占用
netstat -ano | findstr :5432
netstat -ano | findstr :6379

# 停止占用端口的进程，或修改docker-compose.yml中的端口映射
```

---

### 2. 数据库连接失败

**问题**：`Error: connect ECONNREFUSED 127.0.0.1:5432`

**解决**：
```bash
# 确认PostgreSQL容器运行
docker ps | grep postgres

# 查看容器日志
docker logs lobster-postgres

# 重启容器
docker-compose restart postgres
```

---

### 3. Redis连接失败

**问题**：`Error: Redis connection refused`

**解决**：
```bash
# 检查Redis容器
docker ps | grep redis

# 测试连接
redis-cli -h localhost -p 6379 ping

# 重启Redis
docker-compose restart redis
```

---

### 4. Agent API调用失败

**问题**：`OpenRouter API返回401`

**解决**：
```bash
# 检查.env文件
cat .env | grep OPENROUTER_API_KEY

# 确保API Key正确配置
OPENROUTER_API_KEY=sk-or-v1-xxxxx

# 测试API Key（在浏览器或curl）
curl -X POST https://openrouter.ai/api/v1/chat/completions \
  -H "Authorization: Bearer YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"anthropic/claude-3-sonnet","messages":[{"role":"user","content":"test"}]}'
```

---

### 5. WebSocket连接失败

**问题**：前端无法连接WebSocket

**解决**：
```bash
# 检查CORS配置
# .env文件中：
CORS_ENABLED=true
CORS_ORIGIN=*  # 开发环境使用，生产环境改为具体域名

# 检查防火墙
# Windows: 允许端口3000
netsh advfirewall firewall add rule name="Lobster Dev" dir=in action=allow protocol=TCP localport=3000
```

---

### 6. 依赖安装失败

**问题**：`npm install`报错

**解决**：
```bash
# 清除缓存
npm cache clean --force

# 删除node_modules和package-lock.json
rm -rf node_modules package-lock.json

# 使用国内镜像（如果在中国）
npm config set registry https://registry.npmmirror.com

# 重新安装
npm install
```

## 🧰 开发脚本

### 常用命令

```bash
# 启动开发服务器（热重载）
npm run start:dev

# 启动生产模式
npm run start:prod

# 运行单元测试
npm run test

npm run test:e2e

# 测试覆盖率
npm run test:cov

# 代码格式化
npm run format

# ESLint检查
npm run lint

# 数据库迁移
npm run migration:run
npm run migration:revert

# 生成迁移文件
npm run migration:generate -- src/database/migrations/MigrationName
```

### Docker命令

```bash
# 启动所有服务
docker-compose up -d

# 停止所有服务
docker-compose down

# 查看日志
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f postgres

# 重启服务
docker-compose restart

# 清理所有数据（谨慎使用！）
docker-compose down -v
```
