## 详细技术选型

### 1. Web框架

#### ✅ 最终选择：**NestJS 10+**

**选择理由：**

| 优势 | 说明 |
|------|------|
| 🎯 **Java开发者友好** | 与Spring Boot架构高度相似，采用依赖注入、装饰器模式 |
| 🏗️ **企业级架构** | 模块化设计、清晰的代码组织、SOLID原则 |
| 📦 **开箱即用** | 内置WebSocket、微服务、GraphQL、队列等支持 |
| 📚 **完善文档** | 官方中文文档、丰富的社区资源 |
| 🔧 **CLI工具** | 类似Spring Initializr，快速生成代码骨架 |
| 💪 **TypeScript原生** | 完整类型安全，无需额外配置 |

**核心特性：**

```typescript
// 依赖注入（类似Spring @Autowired）
@Injectable()
export class AgentService {
  constructor(
    private readonly redis: RedisService,
    private readonly db: DatabaseService
  ) {}
}

// 装饰器（类似Java注解）
@Controller('agents')
export class AgentController {
  @Post('chat')
  @UseGuards(AuthGuard)
  async chat(@Body() dto: ChatDto) {
    return this.agentService.processChat(dto);
  }
}
```

**备选方案对比：**

| 框架 | 优势 | 劣势 | 适用场景 |
|------|------|------|---------|
| Express | 极简、生态最大 | 缺乏规范、需手动搭建 | 小型项目 |
| Fastify | 高性能、插件丰富 | 学习曲线稍陡 | 性能敏感应用 |
| Koa | 轻量、中间件优雅 | 过于极简 | 定制化需求高 |

### 2. 实时通信

#### ✅ 最终选择：**Socket.io 4+ (基于NestJS WebSocket)**

**选择理由：**

- ✅ **完美聊天室场景**：房间管理、广播、私聊
- ✅ **自动重连**：网络断开自动恢复
- ✅ **事件驱动**：适合Agent间实时协作
- ✅ **双向通信**：服务端主动推送消息
- ✅ **NestJS原生集成**：使用 `@nestjs/websockets`

**实现示例：**

```typescript
@WebSocketGateway({ namespace: '/chat' })
export class ChatGateway {
  @WebSocketServer()
  server: Server;

  @SubscribeMessage('agent:message')
  handleAgentMessage(@MessageBody() data: AgentMessage) {
    // 广播给所有Agent
    this.server.emit('agent:response', data);
  }
}
```

**备选方案：**
- Server-Sent Events (SSE)：单向推送，不适合双向协作
- gRPC Streaming：过于重量级

### 3. 记忆管理

#### ✅ 最终选择：**多层记忆架构**

**架构设计：**

```
┌─────────────────────────────────────────┐
│        记忆管理层（Memory Layer）        │
├─────────────────────────────────────────┤
│                                         │
│  📦 短期记忆 (Short-term Memory)        │
│     ├─ Redis (会话缓存、5分钟TTL)       │
│     └─ LangGraph State (工作流状态)     │
│                                         │
│  💾 长期记忆 (Long-term Memory)         │
│     ├─ PostgreSQL (对话历史)            │
│     └─ ChromaDB (向量检索)              │
│                                         │
│  🤝 共享记忆 (Shared Memory)            │
│     └─ Agent Context Pool (跨Agent)     │
│                                         │
└─────────────────────────────────────────┘
```

**技术组件：**

| 组件 | 技术 | 用途 | TTL策略 |
|------|------|------|---------|
| **Session Cache** | Redis | 会话上下文 | 5分钟 |
| **Message Queue** | BullMQ (Redis) | Agent任务队列 | 按任务 |
| **Conversation History** | PostgreSQL | 对话历史记录 | 永久 |
| **Vector Search** | ChromaDB | 语义检索 | 永久 |
| **Workflow State** | LangGraph | 工作流状态 | 运行时 |

**LangChain Memory集成：**

```typescript
// 自定义Multi-Agent Memory
export class MultiAgentMemory extends BaseChatMemory {
  constructor(
    private redis: Redis,
    private vectorStore: ChromaDB,
    private sessionId: string
  ) {
    super();
  }

  async loadMemoryVariables(): Promise<any> {
    // 从Redis加载短期记忆
    const shortTerm = await this.redis.get(`session:${this.sessionId}`);
    
    // 从向量库检索相关上下文
    const relevant = await this.vectorStore.similaritySearch(query, 5);
    
    return { history: shortTerm, context: relevant };
  }
}
```

### 4. 上下文持久化存储

#### ✅ 最终选择：**分层存储策略**

**存储架构：**

```
┌─────────────────────────────────────────┐
│         存储层（Storage Layer）          │
├─────────────────────────────────────────┤
│                                         │
│  🗄️ 结构化数据 - PostgreSQL 16+         │
│     ├─ 用户会话                          │
│     ├─ Agent状态                         │
│     ├─ 任务队列                          │
│     ├─ 审计日志                          │
│     └─ pgvector扩展（向量字段）          │
│                                         │
│  🎯 向量数据 - ChromaDB                  │
│     ├─ 代码片段向量                      │
│     ├─ 对话上下文向量                    │
│     ├─ 知识库向量                        │
│     └─ 语义检索索引                      │
│                                         │
│  📁 文件存储 - MinIO (S3兼容)            │
│     ├─ 生成的代码文件                    │
│     ├─ 用户上传附件                      │
│     └─ 系统备份快照                      │
│                                         │
└─────────────────────────────────────────┘
```

**技术选型详解：**

##### A. **PostgreSQL 16+** (主数据库)

**核心特性：**
- ✅ JSONB类型：存储复杂Agent状态
- ✅ pgvector扩展：向量检索（可选方案）
- ✅ 事务支持：ACID保证
- ✅ 时序数据：对话历史按时间检索

**NestJS集成：**

```typescript
// TypeORM Entity定义
@Entity()
export class ChatSession {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @Column('jsonb')
  agentStates: Record<string, any>;

  @Column('text', { array: true })
  messageIds: string[];

  @CreateDateColumn()
  createdAt: Date;
}
```

##### B. **ChromaDB** (向量数据库)

**选择理由：**
- ✅ 轻量级、易部署（Python实现）
- ✅ 开源免费
- ✅ 良好的TypeScript SDK
- ✅ 支持本地/远程部署

**替代方案：**
- **Qdrant**：更高性能，生产级，Rust实现
- **Pinecone**：云服务，付费

**使用场景：**

```typescript
// 代码片段向量化存储
await chromaClient.collection('code_snippets').add({
  ids: ['snippet_1'],
  documents: ['function calculateSum(a, b) { return a + b; }'],
  metadatas: [{ language: 'typescript', agent: 'gpt-1' }]
});

// 语义检索
const results = await chromaClient.collection('code_snippets')
  .query({
    queryTexts: ['如何计算两数之和'],
    nResults: 3
  });
```

##### C. **MinIO** (对象存储)

**选择理由：**
- ✅ S3兼容API（可无缝迁移AWS S3）
- ✅ 自托管、成本低
- ✅ 高性能、分布式

**使用场景：**

```typescript
// 存储生成的代码文件
await minioClient.putObject(
  'generated-code',
  `${sessionId}/${timestamp}.ts`,
  codeBuffer
);
```

### 5. LLM集成

#### ✅ 最终选择：**LangGraph + 自定义Adapter**

**架构设计：**

```typescript
// 统一LLM接口
interface ILLMAdapter {
  generate(prompt: string, context: Context): Promise<string>;
  stream(prompt: string, context: Context): AsyncGenerator<string>;
}

// Gemini Adapter（基于CLI调用）
@Injectable()
export class GeminiAdapter implements ILLMAdapter {
  async generate(prompt: string) {
    // 通过shell调用gemini cli
    const result = await execPromise(`gemini-cli "${prompt}"`);
    return result.stdout;
  }
}

// GPT Adapter（基于Codex风格调用）
@Injectable()
export class GPTAdapter implements ILLMAdapter {
  constructor(private httpClient: HttpClient) {}
  
  async generate(prompt: string) {
    // 类似OpenClaw的调用方式
    return await this.httpClient.post('/v1/chat/completions', {
      model: 'gpt-4',
      messages: [{ role: 'user', content: prompt }]
    });
  }
}
```

#### ✅ 最终选择：**BullMQ (基于Redis)**

**选择理由：**

| 特性 | 说明 |
|------|------|

| 📊 **优先级队列** | Agent任务优先级管理 |
| 👀 **可视化监控** | Bull Dashboard |

**使用场景：**

```typescript
// Agent任务队列
@Processor('agent-tasks')
export class AgentTaskProcessor {
  @Process('code-generation')
  async handleCodeGeneration(job: Job) {
    const { prompt, agentType } = job.data;
    
    // 调用对应Agent
    const result = await this.agentService.generate(agentType, prompt);
    
    return result;
  }
}
```

### 8. 认证授权

#### ✅ 最终选择：**Passport.js + JWT**

**方案：**

- **Passport.js**：认证中间件（NestJS官方推荐）
- **JWT**：无状态Token
- **Refresh Token**：长期会话
- **RBAC**：角色权限控制

**实现：**

```typescript
@Injectable()
export class JwtStrategy extends PassportStrategy(Strategy) {
  constructor() {
    super({
      jwtFromRequest: ExtractJwt.fromAuthHeaderAsBearerToken(),
      secretOrKey: process.env.JWT_SECRET
    });
  }

  async validate(payload: any) {
    return { userId: payload.sub, username: payload.username };
  }
}
```
