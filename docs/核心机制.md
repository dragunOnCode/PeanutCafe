# 记忆

当前后端的记忆机制是“多层记忆叠加”，不是单一存储。核心可以概括为 5 层：

1. 短期会话记忆（Redis）
   ShortTermMemoryService 按 sessionId 存最近消息窗口（TTL + maxSize），键为 memory:short:{sessionId}。
   参考：short-term-memory.service.ts:24
2. 共享状态记忆（Redis）
   SharedMemoryService 存 workspaceState 和每个 agent 的决策快照（同样 TTL）。
   参考：shared-memory.service.ts:52
3. 持久消息记忆（DB + 文件回退）
   ChatService.saveMessage 会持久化消息；UUID session 走 Postgres，非 UUID 走进程内 Map。
   同时写 transcript JSONL，作为日志与回退读取来源。
   参考：chat.service.ts:54 workspace.service.ts:129
4. 语义记忆（向量库 Chroma）
   每条消息入 messages collection；自动摘要入 summaries collection，用于后续语义召回。
   参考：chat.service.ts:297 chroma.service.ts:45
5. 会话图状态记忆（LangGraph Checkpointer）
   图执行状态按 thread_id=sessionId 持久化（MemorySaver 或 PostgresSaver）。

读取链路（取最近历史）
优先级是：短期记忆 Redis -> DB -> 进程内 Map -> transcript JSONL。
