## 消息流转

### Hub-and-Spoke消息路由（借鉴OpenClaw）

```
                    用户A    用户B
                      │      │
                      └──┬───┘
                         │ WebSocket连接
                         ↓
            ┌─────────────────────────┐
            │                         │
            │    API Gateway (Hub)    │
            │                         │
            │  • 唯一的消息路由中心   │
            │  • 管理所有会话         │
            │  • 协调Agent响应        │
            │                         │
            └────────┬────────────────┘
                     │
     ┌───────────────┼───────────────┐
     │               │               │
     ↓               ↓               ↓
┌─────────┐    ┌─────────┐    ┌─────────┐
│ Claude  │    │  Codex  │    │ Gemini  │
│ (Spoke) │    │ (Spoke) │    │ (Spoke) │
└─────────┘    └─────────┘    └─────────┘

特点：
✅ 所有消息通过Hub路由
✅ Agent不直接通信
✅ Hub维护全局状态
✅ Agent无状态（可扩展）
```

```

      │
      ↓

      │
   ┌──┴───┬──────┬──────┐
   │      │      │      │

### 三层记忆架构数据流

```
┌─────────────────────────────────────────────────────┐
│                   消息产生                           │
│          用户: "实现用户登录"                         │
└────────────────────┬────────────────────────────────┘
                     │
                     ↓
┌────────────────────────────────────────────────────────┐
│               Layer 1: Hot State (Redis)               │
│                     实时读写                            │
├────────────────────────────────────────────────────────┤
│                                                        │
│  写入 Redis:                                           │
│  ┌──────────────────────────────────────────────┐    │
│  │ Key: memory:short:session-123                │    │
│  │ Value: [最近20条消息]                        │    │
│  │ TTL: 300秒                                   │    │
│  └──────────────────────────────────────────────┘    │
│                                                        │
│  写入 Redis:                                           │
│  ┌──────────────────────────────────────────────┐    │
│  │ Key: memory:shared:session-123               │    │
│  │ Value: {任务状态, 代码文件索引, 技术栈...}   │    │
│  │ TTL: 3600秒                                  │    │
│  └──────────────────────────────────────────────┘    │
│                                                        │
│  发布消息:                                             │
│  ┌──────────────────────────────────────────────┐    │

│  └──────────────────────────────────────────────┘    │
│            ↓ 所有Agent订阅此Channel                   │
└────────────┬───────────────────────────────────────────┘
             │
             │ (异步写入)
             ↓
┌────────────────────────────────────────────────────────┐
│           Layer 2: Durable State (PostgreSQL)          │
│                     持久化存储                          │
├────────────────────────────────────────────────────────┤
│                                                        │
│  INSERT INTO messages:                                 │
│  ┌──────────────────────────────────────────────┐    │
│  │ id: uuid                                     │    │
│  │ session_id: session-123                      │    │

│  │ content: "实现用户登录"                       │    │
│  │ created_at: 2026-02-18 10:00:00              │    │
│  └──────────────────────────────────────────────┘    │
│                                                        │
└────────────┬───────────────────────────────────────────┘
             │
             │ (异步向量化)
             ↓
┌────────────────────────────────────────────────────────┐
│           Layer 3: Vector Store (ChromaDB)             │
│                     语义检索                            │
├────────────────────────────────────────────────────────┤
│                                                        │
│  向量化并存储:                                          │
│  ┌──────────────────────────────────────────────┐    │
│  │ Collection: messages                         │    │
│  │ Document: "实现用户登录"                      │    │
│  │ Embedding: [0.123, -0.456, ...]              │    │
│  │ Metadata: {sessionId, role, timestamp}       │    │
│  └──────────────────────────────────────────────┘    │
│                                                        │
└────────────┬───────────────────────────────────────────┘
             │
             │ (同步写入)
             ↓
┌────────────────────────────────────────────────────────┐
│      Layer 4: File System (借鉴OpenClaw Workspace)     │
│                     审计日志                            │
├────────────────────────────────────────────────────────┤
│                                                        │
│  追加到 transcripts.jsonl:                             │
│  ┌──────────────────────────────────────────────┐    │
│  │ {"timestamp":"...","type":"message",         │    │
│  │  "role":"user","content":"实现用户登录"}      │    │
│  └──────────────────────────────────────────────┘    │
│                                                        │
│  特点：                                                 │
│  • 只追加（Append-only）                               │
│  • JSONL格式（每行一个JSON）                           │
│  • 永久保存                                            │
│  • 人类可读                                            │
│                                                        │
└────────────────────────────────────────────────────────┘

      ↓
┌────────────────────────┐
│  流式生成 (每50ms推送)  │
└────┬───────────────────┘
     │
     ↓ WebSocket: agent:stream
┌────────────────────────┐
│  前端实时显示           │
│  "我来设计登录系统..."  │
│  "首先需要..."         │

     │
     ↓ 生成完成
┌─────────────────────────────────────┐
│  保存完整响应                        │

│  ├─→ Redis (短期记忆)               │
│  ├─→ File System (如包含代码文件)   │
│  └─→ ChromaDB (向量化)              │

## 性能优化策略

### 多级缓存架构

```
请求流:
用户查询 → L1缓存(内存) → L2缓存(Redis) → L3存储(PostgreSQL)
          ↑ 10ms        ↑ 50ms         ↑ 200ms
          │             │              │
          └─ 命中率70%  └─ 命中率25%   └─ 命中率5%
```

**实现**：

```typescript
async function getWithCache(key: string) {
  // L1: 进程内存（最快）
  if (memoryCache.has(key)) {
    return memoryCache.get(key);
  }
  
  // L2: Redis（快）
  const redisValue = await redis.get(key);
  if (redisValue) {
    memoryCache.set(key, redisValue); // 回填L1
    return redisValue;
  }
  
  // L3: 数据库（慢）
  const dbValue = await database.findOne(key);
  if (dbValue) {
    await redis.setex(key, 300, dbValue); // 回填L2
    memoryCache.set(key, dbValue);        // 回填L1
  }
  
  return dbValue;
}
```

### 并行处理

```
┌─────────────────────────────────────────┐
│       新消息到达                         │
└────────────┬────────────────────────────┘
             │
    ┌────────┼────────┐
    ↓        ↓        ↓
  [DB写]  [广播]  [决策]  ← 三个操作并行
    │        │        │
    └────────┼────────┘
             │
             ↓
       [等待最慢的]
             │
             ↓
         [完成]

总耗时 = max(DB写, 广播, 决策)
而非: DB写 + 广播 + 决策
```
