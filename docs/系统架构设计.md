## 架构概述

### 设计理念

Lobster采用**中心化网关 + 分布式Agent**的混合架构，借鉴OpenClaw的Hub-and-Spoke模式，同时融入群聊式交互的创新设计。

**核心特点**：
- 🎯 **单一网关**：所有WebSocket连接、会话管理集中在Gateway
- 🤖 **自主Agent**：Agent独立决策、主动响应
- 💾 **共享状态**：Redis + PostgreSQL统一状态管理
- 📁 **文件持久化**：工作空间使用文件系统存储（借鉴OpenClaw）
- 🔌 **插件化**：Agent、Memory、Channel均可扩展

### 与OpenClaw的异同

| 维度 | OpenClaw | Lobster（本项目） | 借鉴原因 |
|------|----------|------------------|---------|
| **网关架构** | ✅ Hub-and-Spoke | ✅ 采用 | 中心化管理清晰 |
| **Agent模式** | 单Agent多Channel | 多Agent群聊 | ✅ 创新点：群聊式协作 |
| **工作空间** | ✅ 文件系统持久化 | ✅ 采用（增强版） | 可审计、易调试 |
| **热重载** | ✅ Config Hot-reload | ✅ 采用 | 开发友好 |
| **Cron服务** | ✅ 内置调度 | ✅ 采用（BullMQ） | 定时任务必需 |
| **发现服务** | mDNS/Bonjour | ❌ 不需要 | 本项目无本地网络需求 |
| **Channel插件** | 多平台（WhatsApp等） | WebSocket为主 | 本项目专注Web |
| **Memory系统** | 文件+向量库 | ✅ Redis+PostgreSQL+向量库 | 增强实时性 |

## 架构设计原则

### 1. Keep It Simple, Stupid (KISS)

遵循OpenClaw和业界最佳实践：
- ✅ 先用最简单方案解决问题
- ✅ 单一职责原则（每个Agent专注一件事）
- ✅ 工具优先（Tool-first）设计
- ✅ 避免过度抽象

### 2. 中心化控制 + 分布式执行

**中心化**（借鉴OpenClaw Gateway）：
- 单一WebSocket网关
- 统一会话管理
- 集中式路由

**分布式**：
- Agent独立运行
- 并行决策
- 异步响应

### 3. 状态持久化

**文件系统**（借鉴OpenClaw）：
- 工作空间代码文件（.ts, .vue等）
- 配置文件（agents.config.json）
- 审计日志（transcripts.jsonl）

**数据库**：
- 结构化数据（PostgreSQL）
- 缓存和实时状态（Redis）
- 向量索引（ChromaDB）

### 4. 可扩展性

**四类扩展点**（参考OpenClaw）：
1. **Provider插件**：自定义LLM提供商
2. **Tool插件**：自定义工具能力
3. **Memory插件**：替代存储后端
4. **Channel插件**：新消息平台（未来）

## OpenClaw借鉴分析

### ✅ 值得借鉴的组件

#### 1. Hub-and-Spoke架构（网关模式）

**OpenClaw设计**：
- 单一Gateway进程管理所有连接
- 仅Gateway持有会话状态
- 所有消息路由通过Gateway

**Lobster采用**：
```typescript
// API Gateway作为唯一入口
@WebSocketGateway({ namespace: '/chat' })
export class ChatGateway {
  // 管理所有WebSocket连接
  private sessions: Map<string, Session> = new Map();
  
  // 统一路由消息给Agent
  async routeMessage(message: Message) {
    const agents = await this.getAvailableAgents();
    await this.broadcastToAgents(agents, message);
  }
}
```

**优势**：
- ✅ 清晰的单点控制
- ✅ 简化会话管理
- ✅ 易于监控和调试

#### 2. 文件系统工作空间

**OpenClaw设计**：
- 配置、提示、记忆存储为Markdown/JSONL
- 可审计、人类可读

**Lobster采用**：

```typescript
      console.log('配置文件已更新，重新加载Agent...');

      this.eventEmitter.emit('agents.reloaded');
```

**优势**：
- ✅ 开发效率高

```typescript
export class CronTaskProcessor {
  @Process('cleanup-sessions')
  @Cron('0 2 * * *') // 每天凌晨2点
```

### ❌ 不需要的OpenClaw组件

#### 1. mDNS/Bonjour发现服务

**原因**：
- OpenClaw需要本地网络自动发现（跨设备）
- Lobster是中心化Web应用，不需要设备发现

**原因**：
- OpenClaw是通用聊天机器人平台
- Lobster专注于Web编码协作，WebSocket即可

#### 3. 高度去中心化的Session管理

**原因**：
- OpenClaw每个Channel可能有独立Session
- Lobster的Session都在一个Web应用内，中心化管理更简单

## 组件详细设计

### 1. API Gateway（核心Hub）

**职责**：
- WebSocket连接管理
- HTTP REST API服务
- 会话状态管理
- 消息路由
- 配置热重载
- 健康检查

**技术栈**：
- NestJS 10+
- Socket.io 4+
- Passport.js（认证）

**关键实现**：

```typescript
@WebSocketGateway({ namespace: '/chat' })
export class ChatGateway implements OnGatewayConnection, OnGatewayDisconnect {
  constructor(
    private readonly chatService: ChatService,
    private readonly agentService: AgentService,
    private readonly sessionManager: SessionManager
  ) {}
  
  // 连接管理
  async handleConnection(client: Socket) {
    const sessionId = client.handshake.query.sessionId;
    await this.sessionManager.addClient(sessionId, client);
  }
  
  // 消息路由
  @SubscribeMessage('message:send')
  async handleMessage(client: Socket, data: MessageDto) {
    // 1. 保存消息
    const message = await this.chatService.saveMessage(data);
    
    // 2. 广播给所有客户端
    this.server.to(`session:${data.sessionId}`).emit('message:received', message);
    
    // 3. 路由给Agent（异步）
    this.routeToAgents(message, data.sessionId);
  }
  
  // Agent决策协调
  private async routeToAgents(message: Message, sessionId: string) {
    const agents = await this.agentService.getAllAgents();
    const context = await this.sessionManager.getContext(sessionId);
    
    // 并行决策
    const decisions = await Promise.all(
      agents.map(agent => agent.shouldRespond(message, context))
    );
    
    // 处理响应
    agents.forEach((agent, index) => {
      if (decisions[index].should) {
        this.handleAgentResponse(agent, message, sessionId, context);
      }
    });
  }
}
```

### 2. Agent Layer（Spoke）

**设计原则**：
- 单一职责：每个Agent专注一个领域
- 无状态：Agent不保存会话状态
- 插件化：遵循统一接口

**接口定义**：

```typescript
interface ILLMAdapter {
  // 基本信息
  readonly id: string;
  readonly name: string;
  readonly type: string;
  readonly role: string;
  readonly capabilities: string[];
  
  // 核心方法
  generate(prompt: string, context: AgentContext): Promise<AgentResponse>;
  streamGenerate(prompt: string, context: AgentContext): AsyncGenerator<string>;
  shouldRespond(message: Message, context: AgentContext): Promise<Decision>;
```

```typescript
@Injectable()
export class WorkspaceService {
  async saveCodeFile(sessionId: string, file: CodeFile) {
    const workspacePath = `workspace/sessions/${sessionId}`;
    const filePath = path.join(workspacePath, 'code', file.path);
    
    // 1. 保存文件
    await fs.ensureDir(path.dirname(filePath));
    await fs.writeFile(filePath, file.content);
    
    // 2. 记录到transcript
    await this.appendToTranscript(sessionId, {
      type: 'file_created',
      path: file.path,
      author: file.author,
      version: file.version,
      timestamp: new Date()
    });
    
    // 3. 更新元数据
    await this.updateMetadata(sessionId, {
      lastModified: new Date(),
      fileCount: await this.countFiles(sessionId)
    });
    
    // 4. 通知其他Agent（Redis Pub/Sub）
    await this.notifyFileChange(sessionId, file);
  }
  
  private async appendToTranscript(sessionId: string, entry: any) {
    const transcriptPath = `workspace/sessions/${sessionId}/transcripts.jsonl`;
    await fs.appendFile(transcriptPath, JSON.stringify(entry) + '\n');
  }
}
```

```typescript
@Injectable()
export class ConfigWatcherService implements OnModuleInit {
  constructor(
    private readonly agentConfigService: AgentConfigService,
    private readonly eventEmitter: EventEmitter2
  ) {}
  
  async onModuleInit() {
    const watcher = chokidar.watch([
      'config/agents.config.json',
      'config/prompts/**/*.md',
      'config/tools/**/*.json'
    ], {
      persistent: true,
      ignoreInitial: true
    });
    
    watcher.on('change', async (filePath) => {
      console.log(`📝 配置文件变更: ${filePath}`);
      
      if (filePath.includes('agents.config.json')) {
        await this.reloadAgents();
      } else if (filePath.includes('prompts')) {
        await this.reloadPrompts();
      } else if (filePath.includes('tools')) {
        await this.reloadTools();
      }
    });
  }
  
  private async reloadAgents() {
    try {
      await this.agentConfigService.loadAgentsFromConfig();
      this.eventEmitter.emit('agents.reloaded');
      console.log('✅ Agent配置已重新加载');
    } catch (error) {
      console.error('❌ Agent配置重载失败:', error);
    }
  }
}
```

### 6. Background Services

**定时任务**（使用BullMQ）：

```typescript
@Processor('background-tasks')
export class BackgroundTasksProcessor {
  // 每天凌晨2点清理过期会话
  @Process('cleanup-sessions')
  @Cron('0 2 * * *')
  async cleanupExpiredSessions() {
    const expired = await this.sessionRepo.find({
      where: {
        status: 'archived',
        updatedAt: LessThan(new Date(Date.now() - 30 * 24 * 60 * 60 * 1000))
      }
    });
    
    for (const session of expired) {
      await this.archiveToStorage(session);
      await this.sessionRepo.remove(session);
    }
  }
  
  // 每5分钟健康检查
  @Process('health-check')
  @Cron('*/5 * * * *')
  async healthCheck() {
    const agents = await this.agentService.getAllAgents();
    
    for (const agent of agents) {
      const isHealthy = await agent.healthCheck();
      
      if (!isHealthy) {
        // 通知管理员
        await this.alertService.sendAlert({
          type: 'agent_down',
          agent: agent.name,
          timestamp: new Date()
        });
      }
    }
  }
  
  // 每小时生成记忆摘要
  @Process('summarize-memory')
  @Cron('0 * * * *')
  async summarizeMemory() {
    const activeSessions = await this.sessionRepo.find({
      where: { status: 'active' }
    });
    
    for (const session of activeSessions) {
      const shouldSummarize = await this.memoryService.shouldSummarize(session.id);
      
      if (shouldSummarize) {
        const summary = await this.memoryService.generateSummary(session.id);
        await this.memoryService.saveSummary(session.id, summary);
      }
    }
  }
}
```

```typescript
// L1: 内存缓存（快速）
const l1Cache = new Map<string, any>();

// L2: Redis缓存（共享）
const l2Cache = redis;

// L3: 数据库（持久）
const l3Storage = database;

// 多级缓存读取
async function getCached(key: string) {
  // 先查L1
  if (l1Cache.has(key)) return l1Cache.get(key);
  
  // 再查L2
  const l2Data = await l2Cache.get(key);
  if (l2Data) {
    l1Cache.set(key, l2Data); // 回填L1
    return l2Data;
  }
  
  // 最后查L3
  const l3Data = await l3Storage.findOne(key);
  if (l3Data) {
    await l2Cache.setex(key, 300, l3Data); // 回填L2
    l1Cache.set(key, l3Data); // 回填L1
  }
  
  return l3Data;
}
```
