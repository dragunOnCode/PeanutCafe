# 中期审视

## 当前状态速记

- 已完成多 Agent 聊天基础链路：用户消息入会话、决策引擎选 Agent、广播返回结果。
- Codex / Gemini 已具备 CLI 侧会话续聊能力（业务 `sessionId` 映射到各自 CLI 会话 ID）。
- Claude（OpenRouter）已改为服务端多轮组装历史消息，并支持按 token 预算截断上下文。
- 记忆分层基础具备：短期记忆（Redis）、持久化存档（PostgreSQL）、语义检索（Chroma）。

## 已识别问题与风险

1. **会话一致性风险**
   - 不同 Agent 的续聊机制不统一（CLI 会话管理 vs HTTP 历史拼接）。
   - 需要明确“同一轮并发决策时是否共享彼此刚生成的回复”。

2. **观测与排障成本**
   - 仅有状态码时定位困难，需稳定输出上游错误正文（已在 Claude 增强）。
   - 建议统一 Agent 侧日志结构（agentId/sessionId/traceId/model/latency/error）。

3. **语义记忆质量风险**
   - 目前语义召回以消息片段为主，尚未形成“经验教训”结构化知识。
   - 召回质量依赖 embedding 模型与阈值，需持续调参。

## 关键想法（后续方向）

### 1) 引入 Hindsight（复盘记忆）

目标：让 Agent 对“踩过的坑”具备迁移能力。

- 新建 `lessons`（或 `hindsight_events`）语义集合。
- 每条经验沉淀为结构化字段：
  - `problem`（现象）
  - `root_cause`（根因）
  - `fix`（修复）
  - `guardrail`（防回归约束）
  - `scope`（模块/技术栈/版本）
  - `source`（sessionId/agentId/time）
- 在每次 Agent 生成前附加“相似经验检索”结果，作为提示约束。

### 2) 多 Agent 场景继续保留 RAG

结论：有必要，且应加强。

- 短期记忆解决“刚刚聊过什么”。
- RAG 解决“以前相关经验是什么”。
- 多 Agent 协作下，RAG 还是减少重复试错和知识割裂的核心手段。

### 3) 统一会话抽象层

建议抽象统一接口，屏蔽不同供应商差异：

- `resolveSession(agentId, workspaceSessionId)`
- `continueSession(...)`
- `fallbackToFreshSession(...)`
- `persistSessionBinding(...)`

好处：后续新增 Agent 时复用同一会话语义，降低维护复杂度。

## 近期可执行清单（建议）

2. 在 ContextBuilder 中增加“经验召回”分支，并注入到 Agent Prompt。

4. 明确并实现“同轮跨 Agent 共享策略”：
   - 严格隔离（当前默认）
   - 串行共享（后置 Agent 可见前置 Agent 本轮输出）

## 评估指标（用于阶段复盘）

- Agent 响应成功率 / 超时率 / 错误率。
- 语义召回命中率（被实际引用比例）。
- 重复错误率（同类问题再次出现频率）。
- 平均每轮 token 成本与延迟。

## 新增想法：Agent-to-Agent 协作回路（主程 + Reviewer）

目标：支持类似 `Claude 主程 -> @Codex 评审 -> Claude 修复` 的闭环，减少人工转述与重复沟通。

### 协作模式

- 推荐先落地 `Executor-Critic` 模式：
  1. 主程 Agent 产出改动建议/代码草案
  2. 主程 Agent 自动触发 reviewer（例如 `@Codex`）
  3. reviewer 输出结构化审查结果（阻塞问题/建议问题/风险）
  4. 主程 Agent 根据审查结果修复并总结变更

### 协议建议（最小可行）

- 增加结构化消息类型（避免纯自然语言歧义）：
  - `review_request`
  - `review_result`
  - `revision_result`
- `review_request` 建议字段：
  - `requestId`
  - `fromAgentId`
  - `toAgentId`
  - `objective`
  - `scope`（文件/模块）
  - `artifacts`（关键代码片段、diff 摘要）

### 终止条件

- 当 `blockingIssues = 0` 时结束回路。
- 设置最大回合数（例如 2~3 轮）防止无限循环。
- 设置 token/时间预算上限，超限后强制退出并给出当前结论。

### 与当前架构的关系

- 现有 `@mention`、决策引擎、会话记忆与广播机制已具备基础条件。
- 主要新增内容是：
  - 协作协议（消息结构）
  - 轻量工作流状态机（drafting/reviewing/revising/done）
  - 回路终止策略
