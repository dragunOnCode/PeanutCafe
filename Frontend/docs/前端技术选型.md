# Lobster 前端技术选型（调试优先）

## 1. 背景与目标

当前后端已具备 `NestJS + Socket.io + JWT + REST` 能力，且 WebSocket 事件协议明确。  
本前端工程目标是提供一个可快速联调的调试界面，重点解决：

- 事件链路可视化（收发、时序、异常）
- Agent 流式响应可观测（thinking/stream/end）
- 会话与连接状态可定位（重连、错误、参数）

## 2. 选型原则

- 与后端协议强一致（优先 Socket.io 官方客户端）
- 类型优先（TypeScript 约束事件载荷）
- 快速迭代（低样板、启动快、易扩展）
- 与后端工程隔离（目录、脚本、构建独立）

## 3. 技术栈方案

### 3.1 核心框架

- `Vue 3 + TypeScript + Vite`

理由：
- Vue 3 上手快，单文件组件更适合中小团队快速迭代；
- Composition API 便于拆分实时状态逻辑；
- TypeScript 对 WebSocket 事件类型约束友好。

### 3.2 实时通信与数据层

- `socket.io-client`：与后端 Socket.io 协议一致，支持 namespace、重连、ack
- `Pinia`：管理连接态、会话态、事件日志等实时状态
- `@tanstack/vue-query`：处理 REST 请求缓存、重试、状态管理

### 3.3 UI 层

- `Arco Design Vue`（聊天室 + 控制台场景优先）

理由：
- 视觉风格现代，适合聊天和协作类产品；
- 表格、抽屉、Tag、Tabs、消息提示开箱可用；
- 主题能力较强，便于后续向 Lobe 风格靠拢。

### 3.4 工程质量

- `ESLint + TypeScript`
- 后续补充：`Vitest + Testing Library`（单元/组件），`Playwright`（E2E）

## 5. 与后端协作规范

- 事件协议以 `docs/websocket-events.md` 为当前权威来源；
- 前端仅通过 HTTP/WS 与后端交互，不直接依赖后端运行时代码；
- 连接参数统一约定：`auth.token`（鉴权）+ `query.sessionId/userId`（会话上下文）；
- 前端新增事件类型时，同步更新协议文档。

## 6. 风险与缓解

- 协议漂移：通过共享事件类型定义和联调清单控制
- 流式消息乱序：按 `sessionId + agentId` 分桶聚合处理
- 重连重复订阅：在 socket 生命周期内集中注册与清理监听器

## 7. 版本建议

- 当前阶段：以调试控制台为核心（MVP）
- 下一阶段：补全权限、主题、可观测性（错误追踪/导出回放）
